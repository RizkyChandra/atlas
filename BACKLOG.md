# atlas backlog ledger

Every open graphify PR (350) and issue (273) is folded into the port maximally.
Diffs don't apply across a Python→Rust rewrite, so each item is a **requirement**,
not a merge: it maps to a milestone and (for anything marked `fold`) a Rust test.

**CI gate (M10):** every row with `status: fold` must link a passing test id, or CI
fails. That is the proof that "everything was folded in."

## How this file is built

The full 623-row table is generated by a fan-out pass (agents each read a slice of
`gh pr list` / `gh issue list`, emit rows), then a dedup/merge pass. This file
starts with the bucket contract + seed rows; the generated rows are appended under
"## Ledger".

Regenerate the raw inputs:

```bash
gh pr list    --repo Graphify-Labs/graphify --state open --limit 500 \
  --json number,title,isDraft,url > /tmp/atlas-prs.json
gh issue list --repo Graphify-Labs/graphify --state open --limit 500 \
  --json number,title,labels,url  > /tmp/atlas-issues.json
```

## Buckets

| Bucket | Handling | Milestone |
|---|---|---|
| `bug-fix` | Rust regression test first, then fix in the owning crate | owning M |
| `new-language` | grammar crate or bespoke/regex extractor + fixture + golden | M2 |
| `new-ingest` | new doc/media type (csv/tsv/ipynb/pptx/epub) | M8 |
| `feature` | folded into the owning milestone | owning M |
| `perf` | verify win still applies post-`rayon`; else close moot | — |
| `docs` | re-authored against the Rust code (not ported verbatim) | M10 |
| `moot-in-rust` | Python packaging/version/dep items; skipped with a reason | — |
| `arch-decision` | competing architecture; adopt/adapt/decline recorded | — |
| `drafts/dupes` | skipped unless the underlying issue is real | — |
